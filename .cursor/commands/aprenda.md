# Aprenda com o Bug

Um bug complexo acabou de ser identificado e corrigido.

Quero que você aprenda com este erro e transforme isso em **conhecimento permanente do projeto**.

## Premissas

Antes de qualquer escrita, assuma que:

- Este bug **não é óbvio**
- Ele pode reaparecer sob **variações semelhantes**
- Outros agentes precisam aprender com isso **sem repetir o erro**

---

## Etapas

Execute as etapas abaixo na ordem:

### 1. Análise pós-mortem

- Descreva qual era a **causa raiz real** (não o sintoma)
- Explique por que o bug foi **difícil de detectar**
- Indique quais **suposições implícitas** levaram ao erro

### 2. Generalização do problema

- Extraia o **padrão de erro** (classe de problema)
- Liste outros cenários do sistema onde esse padrão pode ocorrer

### 3. Consolidação de boas práticas

- Defina **regras claras** para não repetir esse erro
- Aponte **sinais de alerta** (code smells, decisões perigosas)

### 4. Documentação orientada a agentes

Gere o conteúdo pronto para ser incorporado em:

- `AGENTS.md` (como instrução de comportamento)
- README técnico relevante (como conhecimento do domínio)

---

> Apenas gere o conteúdo sugerido e indique onde ele deve ser inserido.
